## Add Panels (Boxes)

Do log của suricata gửi dưới dạng json nhưng được bọc trong header của pfsense nên ta cần lệnh truy vấn base như sau:
```
index pfsense suricata # lấy ra index pfsense và có chứa suricata bên trong
| rex field=_raw "(?<json_part>\{.*\})" # lấy toàn bộ nội dung khớp với regex json và đặt tên là json_part
| spath input=json_part # spath giúp xử lí với cấu trúc json
| search event_type=alert # chỉ tìm những alert
```
### Box 1 — Alerts Count
```
index=pfsense suricata 
| rex field=_raw "(?<json_part>\{.*\})" 
| spath input=json_part 
| search event_type=alert 
| stats count
```
![alt text](/assets/img/posts/SOC_LAB3/image-2.png)

### Box 2 — Signature-Based Alert Count
```
index=pfsense suricata 
| rex field=_raw "(?<json_part>\{.*\})" 
| spath input=json_part 
| search event_type=alert 
| stats count by alert.signature ## thống kê theo signature
| rename alert.signature as signature
```

### Box 3 - Top Attacker IPs
(Excludes local/private IPs) Query:
```
index=pfsense suricata 
| rex field=_raw "(?<json_part>\{.*\})" 
| spath input=json_part 
| search event_type=alert 
| where src_ip != "192.168.254.128" # loại bỏ ip WAN của pfsense
| stats count as "Attacker IPs" by src_ip  
| sort - "Attacker IPs"
```

### Box 4 - Severity Mapping Over Time
```
index=pfsense "suricata"
| rex field=_raw "(?<json_part>\{.*\})" 
| spath input=json_part 
| search event_type=alert 
| eval sev_label=case( ## label để dễ nhìn hơn
    'alert.severity'=1,"High",
    'alert.severity'=2,"Medium",
    'alert.severity'=3,"Low",
    true(),"Unknown"
)
| timechart span=1h count by sev_label
```

### Box 5 — Protocol Breakdown
```
index=pfsense suricata 
| rex field=_raw "(?<json_part>\{.*\})" 
| spath input=json_part 
| search event_type=alert 
| eval app=coalesce(app_proto, proto) ## nếu có app protocol thì lấy không thì lấy protcol bình thường (TCP, UDP)
| stats count by app
| sort - count
```

### Box 6 — Recent Alerts (Table)
```
index=pfsense suricata 
| rex field=_raw "(?<json_part>\{.*\})" 
| spath input=json_part 
| search event_type=alert
| eval app=coalesce(app_proto,proto)
| eval sev_label=case(
    'alert.severity'=1,"High",
    'alert.severity'=2,"Medium",
    'alert.severity'=3,"Low",
    true(),"Unknown"
)
| table _time sev_label src_ip dest_ip alert.signature alert.category app http.hostname http.url tạo bảng với các trường này
| sort - time
| head 20
```

### Custom rules suricata 
```
alert tcp any any -> any any (msg:"Nmap SYN Scan Detected"; flags:S; threshold:type threshold, track by_src, count 10, seconds 3; sid:1000001; rev:2;)
```
- tcp any any -> any any: cho phép địa chỉ và port bất kì với giao thức tcp
- flags:S SYN gói bắt tay đầu tiên trong TCP
- threshold: type threshold : tránh bị tràn log
- track by_src, count 10, seconds 3: 1 src ip xuất hiện 10 lần trong 3s thì bị alert
rules được kích hoạt khi có 3 gói SYN được gửi trong 10s

### Custom rule reverse shell
trường hợp thường gặp reverse shell là cổng 4444, câu lệnh whoami, hostname,... Mình sẽ tiến hành custom rule cho suricata 
- flow: established: đã hoàn thành bắt tay TCP
```
alert tcp any any -> any 4444 (msg: "Potential Reverse Shell port 4444"; flow: established, to_server; sid:1000050; rev:1;)
alert tcp any any -> any any (msg: "Discovery Command - whoami"; content: "whoami"; flow:established; sid:1000030; rev:1;)
alert tcp any any -> any any (msg: "Discovery Command - hostname"; content: "hostname"; flow:established; sid:1000031; rev:1;)
```

Thực hiện lại một lần nữa, suricata đã bắt được thành công đúng như signature đã tạo
![alt text](/assets/img/posts/SOC_LAB3/image-18.png)
![alt text](/assets/img/posts/SOC_LAB3/image-19.png)

Tuy nhiên, reverse shell là trường hợp nguy hiểm, cần alert cao nên mình sẽ tăng alert severity cho các trường hợp này
- Nmap Syn scan: classtype là attempted-recon
- Reverse shell: classtype: trojan-activity
- Command whoami và hostname: classtype là policy-violation

```
alert tcp any any -> any any (msg:"Nmap SYN Scan Detected"; flags:S; threshold:type threshold, track by_src, count 10, seconds 3; classtype:attempted-recon; sid:1000001; rev:2;)
alert tcp any any -> any 4444 (msg:"Critical Reverse Shell Established"; flow:established,to_server; classtype:trojan-activity; sid:1000050; rev:2;)
alert tcp any any -> any any (msg:"Discovery Command - whoami"; content:"whoami"; flow:established,from_server; classtype:policy-violation; sid:1000030; rev:2;)
alert tcp any any -> any any (msg:"Discovery Command - hostname"; content:"hostname"; flow:established,from_server; classtype:policy-violation; sid:1000031; rev:2;)
```

Ta đã ghi hận các signature trên với severity là high
