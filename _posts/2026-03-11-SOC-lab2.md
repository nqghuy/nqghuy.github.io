## Verify DNS Configuration
Trong Active Directory, máy trạm không tìm Domain Controller bằng IP trực tiếp mà thông qua DNS record.

Ví dụ domain: soclab.local

Khi một máy Windows join domain hoặc đăng nhập, luồng xử lý sẽ là:
Client -> DNS query -> tìm Domain Controller

Ví dụ truy vấn:
_ldap._tcp.dc._msdcs.soclab.local

DNS sẽ trả về địa chỉ của Domain Controller, ví dụ 192.168.10.10.

## Tạo VPN Access trên pfSense với xác thực Active Directory
Phần này sẽ cấu hình VPN trên pfSense để người dùng đăng nhập bằng tài khoản AD.
Mục tiêu là cho phép truy cập từ bên ngoài vào mạng nội bộ như đang ở LAN, đồng thời vẫn quản lý quyền truy cập tập trung bằng AD.

Tạo 1 LDAP Authentication Server
Auth server trỏ về Domain Controller.
- Tên SOC-LDAP
- IP là DC IP 
- Port 389 cho LDAP (636 LDAPS)
- Auth container là CN=Users,DC=soclab,DC=local (pfSense sẽ tìm kiếm ở đây trước)
- Bind credentials: CN=pfsense bind,CN=Users,DC=soclab,DC=local pfSense dùng tài khoản này để truy vấn user khác cần xác thực

## Creating the OpenVPN Server
- Description : Soclab_openvpn
- Server Mode: Remote Access (SSL/TLS + User Auth)
- Backend for Authentication: SOC-LDAP (Since we are going to use AD for authentication)

## Set Up Dynamic DNS (DDNS) on pfSense
Đa phần mạng internet gia đình dùng IP động. Nghĩa là sau mỗi lần modem reset hoặc theo chu kỳ, ISP có thể đổi IP public của bạn.
DDNS giúp giải quyết vấn đề này bằng cách gắn một hostname cố định (ví dụ: mysoclab.ddns.net) với IP public hiện tại.

Cơ chế hoạt động: mỗi khi IP thay đổi, pfSense sẽ tự cập nhật IP mới lên nhà cung cấp DDNS.

![alt text](/assets/img/posts/SOC_LAB2/image-84.png)
![alt text](/assets/img/posts/SOC_LAB2/image-85.png)
